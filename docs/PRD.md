# 讀書筆記本系統 - 產品需求文件 (PRD)

## 1. 專案概述

### 1.1 背景與動機
圖書館中往往有大量的藏書，但不易快速了解每本書的熱門程度與讀者真實的反饋。為了解決這個問題，我們需要建立一個「讀書筆記本系統」，給予圖書館一套快速歸納書本歡迎程度的機制，同時也讓讀者能夠快速得知暢銷度，進而提升圖書館的借閱率與管理效率。

### 1.2 目標用戶
- 學校圖書人員（主要受眾與管理者）
- 讀者（一般使用者）

### 1.3 核心價值主張
- **對圖書人員**：快速掌握書籍受歡迎程度、追蹤圖書庫存狀態與讀者推薦（二度購買需求），優化館藏採購與管理。
- **對讀者**：能藉由平台了解書籍暢銷度與其他讀者的心得評分，找到好書並分享心得。

## 2. 功能需求

以下列出系統的主要功能，並以使用者故事（User Story）格式描述：

1. **紀錄書名**
   - 作為 **圖書人員 / 讀者**，我希望 **能夠在系統中新增與查詢書籍名稱**，以便 **建檔或尋找想要閱讀/管理的特定書籍**。
2. **記錄心得**
   - 作為 **讀者**，我希望 **能夠為閱讀過的書籍紀錄讀書心得**，以便 **分享給其他人並作為未來的選書參考**。
3. **讀者評分**
   - 作為 **讀者**，我希望 **能夠針對閱讀過的書籍給予評分（例如 1-5 星）**，以便 **讓圖書館與其他讀者快速得知該書的整體受歡迎程度與暢銷度**。
4. **圖書庫存**
   - 作為 **學校圖書人員**，我希望 **能夠在系統中紀錄與更新每本書的現有庫存數量**，以便 **追蹤借閱狀況並確保書籍妥善管理**。
5. **圖書二度購買需求**
   - 作為 **學校圖書人員**，我希望 **能夠蒐集並查看讀者提出對特定熱門/缺貨書籍的二度購買需求（進書推薦）**，以便 **根據真實的讀者需求進行未來的採購規劃**。

## 3. 非功能需求

### 3.1 技術限制
- 架構：Web 應用程式
- 後端框架：Flask (Python)
- 模板引擎：Jinja2
- 資料庫：SQLite (適合初期 MVP 輕量化部署)

### 3.2 效能與安全考量
- **效能**：頁面載入時間應低於 2 秒，查詢功能必須在大量書目下保持流暢。
- **安全**：
  - 需防範 SQL Injection 與 XSS 攻擊（確保心得留言無惡意語法）。
  - 對圖書人員（庫存與採購管理）應具備基本的權限控管與登入機制。

## 4. MVP 範圍

### Must Have (必須擁有)
- 書籍基本資料建檔與查詢功能（書名、作者等）。
- 讀者留下心得與 1-5 星評分系統。
- 首頁顯示整體評分最高、最受歡迎的書籍排行（暢銷度呈現）。
- 圖書庫存基本欄位顯示與管理（新增/修改庫存量）。

### Should Have (應該擁有)
- 讀者一鍵點擊「推薦購買／提醒補貨」功能，納入二度購買需求。
- 專屬圖書館員的後台，能統整查看高評分及高需求購買的書籍清單。

### Nice to Have (可以擁有)
- 讀者間的心得交流互動（留言、按讚）。
- 整合實體圖書館現有的條碼借閱系統 API。
